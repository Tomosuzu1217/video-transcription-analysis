1. アプリケーション概要

目的

動画CM（コマーシャル）の書き起こし・分析・マーケティング最適化を一元的に行うWebアプリケーション。 動画/音声ファイルをアップロードし、OpenAI Whisperで自動文字起こしを行い、NLP（形態素解析）およびGoogle Gemini AIによる多角的なコンテンツ分析を実施する。広告実績データ（Excel）と動画を紐付けて、事業貢献度の可視化と改善提案を提供する。

主な利用シナリオ

  • CM動画のアップロードと自動文字起こし（Whisper large-v3モデル使用）
  • 文字起こし結果のセグメント単位での確認・編集・エクスポート（SRT/TXT/JSON）
  • キーワード頻度分析、キーワード×コンバージョン相関分析
  • Gemini AI による心理分析、ランキング比較分析、マーケティングレポート生成
  • Excel広告実績データのインポートと動画コードによる紐付け
  • 事業貢献スコアに基づく動画ランキングと媒体別パフォーマンス可視化
  • マーケティングダッシュボードでのKPI概要・レポート・PPTX出力

対象ユーザー

マーケティングチーム、動画制作チーム、広告運用担当者。共有パスワード認証によるチーム共有型。

3. システムアーキテクチャ

全体構成

ハイブリッド構成: フロントエンドは Firebase Hosting + Supabase（DB/Storage/Realtime）に直接接続。バックエンドはローカルで動作し、Whisper による文字起こしとNLP処理を担当。AI分析はフロントエンド・バックエンド双方からGemini APIを呼び出す。

フロントエンド (React SPA)

  • Firebase Hosting で配信
  • Supabase JS で DB/Storage 直接操作
  • Gemini API で AI 分析（フロント側）
  • Supabase Realtime で動画状態監視
  • PasswordGate による共有パスワード認証

バックエンド (FastAPI)

  • ローカルサーバー（開発/社内利用）
  • Whisper 音声文字起こし
  • fugashi 形態素解析 (NLP)
  • Gemini API AI分析
  • 動画アップロード・ストリーミング
  • サムネイル自動生成 (ffmpeg)

データベース

  • Supabase PostgreSQL（本番）
  • SQLite（バックエンドローカル）
  • RLS: 全テーブル anon 許可
  • Realtime: videos テーブル有効

ストレージ

  • Supabase Storage: videos バケット
  • Signed URL（4時間有効、3.5時間キャッシュ）
  • ローカル: uploads/ ディレクトリ
  • 音声のみ保存モード対応

6. API仕様

バックエンドAPI: http://localhost:8000/api をプレフィックスとするREST API。 フロントエンドは Supabase JS を介してDBに直接アクセスする部分と、バックエンドAPIを呼び出す部分のハイブリッド構成。

6.1 動画 (Videos)

POST /api/videos/upload 動画/音声ファイルをアップロード（最大20ファイル同時、最大2048MB/ファイル）。ffprobeでメタデータ抽出、STORE_AUDIO_ONLYモードではffmpegで音声抽出。アップロード後に文字起こしキューへ自動投入。
GET /api/videos 動画一覧取得（ページネーション: page, per_page）。created_at DESC 順。

  • パスワードは SHA-256 ハッシュ化して環境変数に保存
  • セッションは authSession.ts で管理

Pure ASGIミドルウェアで以下のヘッダーを付与:

  • X-Frame-Options: DENY
  • X-XSS-Protection: 1; mode=block
  • Referrer-Policy: strict-origin-when-cross-origin
  • Permissions-Policy: camera=(), microphone=(), geolocation=()

※ストリーミング・サムネイルエンドポイントはスキップ

CORS

  • 許可オリジン: 環境変数 CORS_ORIGINS（デフォルト: http://localhost:5173）
  • 許可メソッド: GET, POST, PUT, PATCH, DELETE, OPTIONS
  • クレデンシャル: 許可

入力バリデーション

データ保護

  • Supabase RLS: 全テーブルで有効（現在は anon 全許可ポリシー）
  • APIキー: Supabase DB にはマスク表示、フロントエンド側は暗号化保存 (crypto.ts)
  • Supabase Storage: Signed URL（4時間有効期限）でアクセス制御
